History

Kotlin was originally unveiled in July 2011 by JetBrains. However, it was not released until February 15, 2016. Shortly after at Google I/O 2017, "Google announced first-class support for Kotlin on Android"(1). By May 7, 2019, Google announced that Kotlin was it's preferred language for Android developers.

Design

Kotlin was designed to be an improved version of Java for mobile development, as it is objected-oriented and is fully interoperable with Java code. Kotlin classes tend to be much more concise than Java. For example, classes that soley hold data in Java require plenty of "boilerplate code" (ex. getters, setters, equals...) whereas Kotlin allows the data keyword to handle everything in the compiler.

Coroutines

For single-threaded systems (like Android), Java's solution is to make multiple background threads that can become complicated and hard to handle errors. Kotlin uses stackless threads, called coroutines, that use less memory to achieve code that appears synchronous.

Higher-Order Functions

Kotlin supports passing functions as parameters or returning functions from other functions. They can also be stored in data structures or variables

Objects

Every function in Kotlin is an object. Primitive data types are objects.

Inline Functions

Inline functions can be used to reduce the overhead created by allowing higher-order functions
